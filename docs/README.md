# OctoAcme Project Management Documentation

Welcome to the OctoAcme Project Management Docs. This folder contains comprehensive guidance for how we plan, execute, and deliver projects at OctoAcme.

## Overview

OctoAcme follows a structured, customer-focused approach to project management designed to scale institutional knowledge and enable consistent, repeatable project execution. Our methodology prioritizes clear ownership, data-informed decisions, and psychological safety.

### Core Principles

- **Customer-first**: Prioritize customer value and usability in all decisions
- **Iterative delivery**: Deliver small, testable increments to enable fast feedback
- **Clear ownership**: Each project has named responsible parties with defined roles
- **Data-informed decisions**: Measure impact and iterate based on evidence
- **Psychological safety**: Encourage feedback, learning, and continuous improvement

## Project Lifecycle

Every OctoAcme project follows these five phases:

1. **Initiation** - Validate business need, align stakeholders, create one-pager, make go/no-go decision
2. **Planning** - Break down work, estimate scope, identify dependencies, create actionable backlog
3. **Execution & Tracking** - Build, test, review, iterate with daily standups and weekly syncs
4. **Release & Deployment** - Deploy to production with standardized safety checks and rollback plans
5. **Retrospective & Improvement** - Capture learnings and convert into actionable improvements

## Key Roles

OctoAcme projects operate with clear role assignments to ensure accountability and alignment:

- **Project Manager (PM)**: Coordinates delivery, manages schedules, risks, and communications
- **Product Manager (PdM)**: Defines outcomes, prioritizes the backlog, and measures success
- **Developers**: Implement features, collaborate on design, write tests and documentation
- **QA/Testing**: Validate quality, verify acceptance criteria, and execute test plans
- **Stakeholders**: Provide inputs, approvals, and business context

See [Roles & Personas](./octoacme-roles-and-personas.md) for detailed descriptions of each role.

## Key Artifacts

Every OctoAcme project maintains these artifacts:

- **Project Charter / One-pager** - Problem statement, goals, success metrics, stakeholders, timeline
- **Roadmap and Release Plan** - Milestones, release schedule, dependencies
- **Sprint/Iteration Backlog** - Prioritized backlog with acceptance criteria and estimates
- **Risk Register** - Identified risks, impact, likelihood, mitigation plans
- **Definition of Done** - Quality standards that must be met before work is complete
- **Retrospective Notes** - Learnings and action items captured after milestones

## Communication Cadence

- **Daily Standups** (15 min) - Team shares progress, blockers, and dependencies
- **Weekly PM/PdM Sync** - Alignment on priorities, risks, and decisions
- **Weekly Stakeholder Updates** - Status, progress toward metrics, and flagged risks
- **Sprint/Milestone Reviews** - Demo of completed work and feedback collection
- **Monthly Executive Updates** - High-level progress and key milestones
- **Ad-hoc Escalations** - Critical issues follow defined escalation paths
